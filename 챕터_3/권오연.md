## Ch 3. 액션과 계산, 데이터의 차이를 알기

- 액션
  - 실행 시점과 횟수에 의존
  - 부수 효과가 있는 함수
  - 순수하지 않은 함수
  - ex) 이메일 보내기, 데이터베이스 읽기
- 계산
  - 순수 함수
  - 수학 함수
  - ex) 최댓값 찾기, 이메일 주소가 올바른지 확인하기
- 데이터
  - 이벤트에 대한 사실
  - ex) 이메일 주소
- 실생활 예시
  - 냉장고 확인하기 - 액션
  - 운전해서 상점으로 가기 - 액션
  - 필요한 것 구입하기 - 액션
  - 운전해서 집으로 오기 - 액션
- 계산은 해석이 필요 없지만, 데이터는 해석이 필요
- 쿠폰에 등급매기기가 데이터?
- 일반적인 구현 순서
  1. 데이터
  2. 계산
  3. 액션
- 계산의 장점
  - 테스트 하기 쉽다.
  - 기계적인 분석이 쉽다.
  - 조합하기 좋다.
  - 함수형 프로그래머가 좋아하는 것
- 액션은 코드 전체로 퍼진다.
  - 액션 함수를 다른 함수에서 사용하면 그 함수도 액션이 됨
